# 访问dom的次数越多，代码的运行速度越慢。尽量把运算让js处理

# 使用innerHTML和DOM的原生方法来创建dom，不同浏览器的表现速度不一样，建议用innerHTML

# 在相同的内容和数量下，遍历一个数组的速度明显快于遍历一个HTML集合，见demo.js中的函数 domLoopFn

# 函数中读写局部变量是最快的，读写全局变量是最慢的。所以尽量把集合存在局部变量中，见demo.js 中的函数 collectionFn

*** 小结 *** 
+ 最小化DOM访问次数，尽可能在js端处理
+ 如果需要多次访问某个DOM节点，请使用局部变量存储它的引用
+ 处理HTML集合时，在循环中把集合的长度缓存到一个变量中；如果需要经常操作集合，建议把它拷贝到一个数组中
+ 减少重排重绘；批量修改样式时，'离线'操作DOM树，使用缓存，并减少访问布局信息的次数
+ 动画中使用绝对定位，使用拖放代理
+ 使用事件委托来减少事件处理器的数量
